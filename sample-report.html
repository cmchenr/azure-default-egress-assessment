Remediation Guidance

Recommended actions to address identified issues

Remediation Strategies

Timeline: Azure is eliminating their default Egress feature on September 30th, 2025. Plan remediation accordingly. See official documentation

Secure Remediation (Recommended)

User Defined Routes (UDRs):
  • Create route tables with default route (0.0.0.0/0) pointing to Network Virtual Appliance (NVA)
  • Associate route tables with affected subnets
  • Provides security inspection and centralized control
  • Recommended for production environments

Managed Egress (Alternative)

Azure NAT Gateway:
  • Deploy NAT Gateway for managed outbound connectivity
  • Associate with subnets requiring internet access
  • Provides predictable SNAT but no security inspection
  • Suitable for development and testing environments

Special Cases

Mixed-Mode Subnets:
  • Separate resources with public IPs from those without
  • Create dedicated subnets for public-facing resources
  • Implement consistent connectivity strategy
CIDR Overlaps:
  • Address overlapping CIDR ranges before VNET Hub or Virtual WAN hub connectivity
  • Consider VNet peering as alternative
  • Plan IP address space redesign if necessary

Remediation Priority

Priority Order:
  1. High: Subnets with "Default Egress" classification
  2. Medium: "Mixed-Mode" subnets requiring standardization
  3. Low: VNets with CIDR overlaps (impacts future connectivity)
